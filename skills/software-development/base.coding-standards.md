# 代码规范

## 命名
| 类型 | 规范 |
|------|------|
| 变量/函数 | camelCase |
| 类/组件 | PascalCase |
| 常量 | UPPER_SNAKE_CASE |
| 文件 | PascalCase |

## 目录结构原则
- 功能域划分：按功能域划分，不按文件类型平铺
- 模块自包含：业务模块自包含，不跨模块直接引用内部文件
- 公共提升：可复用内容提升到公共层
- 结构约定：具体目录结构在 `03-pre-development` 阶段约定

## 强制规则
- 常量定义：常量单独定义，禁止魔法数字和魔法字符串
- 复用提取：重复逻辑/样式提取为公共函数/组件/样式变量，不允许复制粘贴
- 参数限制：参数超过 3 个改用对象传参
- 提交清洁：不提交 console.log、堆积的 TODO
- 错误处理：所有错误必须有明确处理，禁止静默失败（空 catch、吞掉异常）
- 注释原则：注释说明"为什么"，不说"做什么"
