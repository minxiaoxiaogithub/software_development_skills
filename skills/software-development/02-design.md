# 软件设计

## 目的
将需求转化为可执行的模块设计方案，明确模块边界、接口和 UI 交互。

## 输入
- 需求文档：`01-requirements` 输出的需求清单 + 范围边界说明

## 步骤

- 分批设计：设计拆成批次，每批次与开发者 review 确认，记录设计决策
- 批次内容：每批次完成模块边界、接口（入参/出参/错误）、数据模型、UI 交互流程、异常流程
- 写入文档：按输出要求写入文档（路径遵守 `00-overview` 文档存放规范）
- 执行 review：遵守 `base.review-base`，通过后才进入下一阶段
- 提交：review 通过后按 `base.git-workflow` 提交

## 输出
- 模块划分方案：职责、边界、接口定义
- UI 交互流程：覆盖所有交互状态
- 设计决策记录

## Review 检查清单
> 通用规则见 `base.review-base`

- 模块职责：单一，边界清晰，无职责重叠
- 接口定义：模块间接口明确，数据流向清晰
- 需求覆盖：覆盖所有已确认的需求，无遗漏
- 公共模块：已识别，无重复设计
- UI 流程：覆盖所有交互状态（空态、加载、错误、成功）
