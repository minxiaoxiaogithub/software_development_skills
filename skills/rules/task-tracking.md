# 任务追踪

## 状态标记

| 状态 | 标记 |
|------|------|
| 待处理 | ⬜ |
| 已完成 | ✅ |
| 已跳过 | ⏭️ |
| 出错 | ❌ |

## 规则

- 创建：每次执行多步骤任务时，在 `docs/tasks/doing/` 下创建 `task-<类型>-<时间戳>.md`
- 格式：第一行写 `进度：0/<总步骤数>`，后续列出所有步骤，每步前加 ⬜
- 实时更新（强制）：每完成一个步骤，必须立即更新任务文件，禁止批量更新
- 子任务：子任务用缩进表示，同样使用状态标记
- 完成：全部步骤处理完后，将文件移到 `docs/tasks/done/`

## 示例

```markdown
进度：2/4

- ✅ 需求整理
- ✅ 需求确认
- ⬜ 范围界定
- ⬜ 写入文档
```
